# Rust Worker с Celery Bridge

## Описание
Этот воркер написан на Rust и использует мост для взаимодействия с Celery.
Поскольку Rust не имеет встроенной поддержки Celery, мы используем промежуточный
Python скрипт для моста между Celery и Rust приложением.

## Архитектура
- Celery воркер на Python (celery_bridge.py) получает задачи из очереди
- Python мост вызывает Rust приложение через subprocess
- Rust приложение обрабатывает задачу и возвращает результат
- Python мост отправляет результат обратно в Celery

## Поддерживаемые операции
Rust воркер поддерживает следующие математические операции:
- `multiply` - умножение значения на множитель (по умолчанию 2.0)
- `add` - сложение значения с слагаемым (по умолчанию 10.0)
- `power` - возведение значения в степень (по умолчанию 2.0)
- `sqrt` - извлечение квадратного корня из значения
- По умолчанию - удвоение значения

## Зависимости
- Python 3.8+
- Celery
- redis
- Rust 1.65+

## Установка и запуск

1. Сборка Rust приложения:
```bash
cargo build --release
```

2. Запуск Celery воркера:
```bash
celery -A celery_bridge worker --loglevel=info
```

## Как это работает
1. Celery воркер получает задачу из очереди
2. Вызывается функция process_rust_math_task
3. Python subprocess вызывает Rust приложение с параметрами задачи
4. Rust приложение обрабатывает задачу и возвращает результат в stdout
5. Python мост парсит результат и возвращает его в Celery

## Формат задачи
Задача должна быть в формате JSON со следующими полями:
```json
{
  "task_id": "unique_task_id",
  "value": 5.0,
  "operation": "multiply",
  "multiplier": 3.0
}
```

Дополнительные поля будут переданы в результат без изменений.
